    配置标准 TCP/IP 打印机端口

    语法 

    cscript prnport.vbs -t -r PortName [-s RemoteComputer] [-o {raw -n PortNumber | lpr}]
 [-h IPAddress] [-q QueueName] [-m{e | d}] [-i IndexName] [-y CommunityName] [-2{e | d}] [-u UserName -w Password]

    参数

    -t

    必需。指定要配置标准 TCP/IP 打印机端口。

    -r PortName

    必需。指定连接打印机的端口。

    -s RemoteComputer

    按名称指定要配置其端口的远程计算机。如果不指定计算机，则配置本地计算机的端口。

    -o {raw -n PortNumber | lpr}

    指定端口使用的协议：TCP raw 或 TCP LPR。如果使用 TCP raw，则指定 TCP raw 打印机端口的端口号。默认情况下，该端口号为 9100。详细信息，请参阅“”。多数打印机使用 TCP raw。在 UNIX 网络上，打印机端口通常使用 TCP LPR。有关 TCP raw 的详细信息，请参阅“”。有关 TCP LPR 的详细信息，请参阅 RFC 编辑器网站上的 RFC 1179。

    -?|-h

    按 IP 地址指定要配置端口的打印机。

    -q QueueName

    指定 TCP raw 端口的队列名称。

    -m{e | d}

    指定是否启用 SNMP。参数 e 可以启用 SNMP。参数 d 可以禁用 SNMP。

    -i IndexName

    指定 SNMP 索引（如果启用了 SNMP）。详细信息，请参阅 RFC 编辑器网站上的 RFC 1759。

    -y CommunityName

    指定 SNMP 社区名（如果启用了 SNMP）。有关详细信息，请参阅“”。

    -2{e | d}

    指定是否启用 TCP LPR 端口的双重后台打印（也称二次后台打印）。需要双重后台打印是因为 TCP LPR 必须在发送到打印机的控制文件中包含精确的字节数计数，但协议却不能从本地打印提供程序那里获得该计数。

    因此，当文件以后台方式发送到 TCP LPR 打印队列时，该文件也以后台打印方式存储为 system32 目录下的临时文件。TCP LPR 可以确定临时文件的大小并将该大小发送到运行 LPD 的服务器。参数 e 可以启用双重后台打印。参数 d 可以禁用双重后台打印。

    -u UserName -w Password

    指定具有权限的帐户（该权限可使用 Windows 管理工具 (WMI) 服务连接到要配置其端口的计算机）。该计算机管理员组的所有成员都具有这些权限，不过也可以将这些权限授予其他用户。如果没有指定帐户，则必须使用具有这些权限的帐户进行登录，才能使用该命令。

    /?

    在命令提示符显示帮助。

    注释

    该命令启动 systemroot\system32 目录下的一个脚本。在该目录为当前目录时须在命令提示符键入该命令，或者必须在 cscript 命令的开始位置键入到该目录的完整路径。

    如果提供的信息包含空格，请使用引号将文本引起来（例如，"Computer Name"）。
