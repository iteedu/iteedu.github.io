    安 装

    Installation

    要使用Tomcat5来开发网站应用程序,你必须先安装它(以及它所以赖的其他软件)。下面的章节里概述了必须的步骤。

    JDK

    Tomcat 5.5被设计适合在J2SE 5.0上运行，但是它也可以在JDK 1.4上运行，请使用与Tomcat 5.5相兼容的软件包，这在Tomcat installation里有详细说明。

    请到http://java.sun.com/j2se/ .去查找和许多操作平台相适宜的JDKs。

    Tomcat

    Binary downloads of the Tomcat server are available from http://tomcat.apache.org/download-55.cgi. This manual assumes you are using the most recent release of Tomcat 5. Detailed instructions for downloading and installing Tomcat 5 are available here.

    在这个手册的后面有shell scripts的样本。 我们假设你设置好了环境变量CATALINA_HOME,它里面的路径(pathname)应该指向已安装的Tomcat5所在目录。

    Ant

    从http://ant.apache.org/bindownload.cgi可以下载Ant建造工具。这个手册假设你使用Ant1.4或其后版本。这里的指示说明应该和Ant1.4以后的版本相适合，但还没有被测试过。

    从上面提到的网址下载和安装Ant。然后，把Ant distribution里的bin directory加入到你的路径(PATH)环境变量里，按常规操作你的系统平台。在完成这些之后，你就可以直接执行ant shell命令了。

    CVS

    除了上面所描述的必须工具以外，我们非常鼓励你去下载和安装源代码控制系统(source code control systems),比如象Concurrent Version Systems(CVS),用来维护管理你的网络应用程序的historical版源代码文件。除了服务器以外，你还需要相应的客户(client)工具来取出(check out)源代码文件,和存入(check in)修改过的版本。

    关于安装和使用源代码控制应用的详细说明超除了着个手册的范围。不过，可以从http://www.cvshome.org .下载适用于许多平台上的CVS服务器和客户工具(以及其文献)。
