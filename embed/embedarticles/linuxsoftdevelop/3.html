    • Linux 基本命令实战

    根据项目需要，需要对Linux内核打入补丁，重新编译，所以系统的基本配置还是必须知道，一些查看系统配置的基本命令就必须用到，可是好久没有了(大概有3个月多了)，都生疏了，在此记录一下子……

    Linux最基础的命令

    uname -r：查看内核版本

    uname -ar：查看内核版本以及操作系统的位数；

    cat /etc/issue：查看操作系统版本；

    cat /proc/cpuinfo：查看CPU信息命令；

    cat /proc/meminfo：查看内存信息命令

    fdisk -l：查看硬盘信息命令

    查看CPU信息—型号 

     # cat /proc/cpuinfo | grep name | cut -f2 -d: | uniq -c

    8 Intel(R) Xeon(R) CPU E5410 @ 2.33GHz

    (看到有8个逻辑CPU, 也知道了CPU型号) 
    # cat /proc/cpuinfo | grep physical | uniq -c

    4 physical id : 0
    4 physical id : 1

    (说明实际上是两颗4核的CPU)

     # getconf  LONG_BIT

    32

    (说明当前CPU运行在32bit模式下, 但不代表CPU不支持64bit)
          # cat /proc/cpuinfo | grep flags | grep ' lm ' | wc -l

    8

    (结果大于0, 说明支持64bit计算. lm指long mode, 支持lm则是64bit)

    再完整看cpu详细信息, 不过大部分我们都不关心而已。 
     # dmidecode | grep 'Processor Information'

    查看内存信息

     # cat /proc/meminfo

    查看当前操作系统内核信息 

     # uname -a

    Linux euis1 2.6.9-55.ELsmp #1 SMP Fri Apr 20 17:03:35 EDT 2007 i686 i686 i386 GNU/Linux

    查看当前操作系统发行版信息

    # cat /etc/issue | grep Linux

    Red Hat Enterprise Linux AS release 4 (Nahant Update 5)

    查看机器型号

     # dmidecode | grep "Product Name" 

    查看网卡信息 

     # dmesg | grep -i eth  

    查看系统内核情况

    其实应该通过Physical Processor ID来区分单核和双核。而Physical Processor ID可以从cpuinfo或者dmesg中找到：

    例如

    root@debian：~：0# dmesg | grep CPU | grep ID

    CPU0： Initial APIC ID： 0， Physical Processor ID： 0

    CPU1： Initial APIC ID： 1， Physical Processor ID： 0

    CPU2： Initial APIC ID： 6， Physical Processor ID： 3

    CPU3： Initial APIC ID： 7， Physical Processor ID： 3

    可以看到，这台机器有两个双核的CPU，ID分别是0和3.
